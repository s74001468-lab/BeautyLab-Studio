# Product Requirements Document (PRD): BeautyLab Studio Bot

## 1. Overview (Обзор)
**Цель проекта**: Разработать Telegram-бота (MVP) для салона красоты BeautyLab Studio (г. Краснодар) для автоматизации записи клиентов, снижения нагрузки на администратора и минимизации пропущенных записей за счет системы напоминаний.
**KPI**: 
- +20% новых записей
- −50% нагрузки на администратора
- −30% пропущенных записей

## 2. User Stories (Пользовательские сценарии)
1. **Запись**: Как клиент, я хочу выбрать услугу, мастера, дату и время, оставить комментарий и записаться, чтобы гарантировать себе место.
2. **Мои записи**: Как клиент, я хочу видеть свои активные записи, чтобы не забыть о них.
3. **Отмена/перенос**: Как клиент, я хочу иметь возможность отменить или перенести запись. Если до записи более 24 часов — всё происходит автоматически в боте. Если менее 24 часов — бот даёт контакты администратора.
4. **Напоминания**: Как клиент, я хочу получать напоминания (за 24 часа, 2 часа и 10 минут), чтобы прийти вовремя.

## 3. Data Model (Модель данных в Google Sheets)
БД состоит из одного листа (Sheet) для MVP.
**Столбцы (Заголовки)**:
* `ID` (Уникальный ID записи, генерируется ботом)
* `Telegram ID` (ID пользователя)
* `Имя клиента` (Вводит клиент)
* `Телефон` (Запрашивается контактом или вводом)
* `Услуга` (Выбранная услуга)
* `Мастер` (Выбранный мастер или "Любой")
* `Дата` (Формат DD.MM.YYYY)
* `Время` (Формат HH:MM)
* `Длительность` (в минутах)
* `Комментарий` (Дополнительные пожелания клиента)
* `Статус` (Active, Cancelled, Completed)
* `Created_at` (Время создания заявки)

## 4. Services & Durations (Услуги и длительность)
- Женская стрижка — 60 минут
- Окрашивание — 90 минут
- Мужская стрижка — 30 минут
- Маникюр / Педикюр — 60 минут
*Мастера*: Анна, Мария, Алексей, "Любой мастер".

## 5. Flows (Логика работы)
* **Запись (Booking Flow)**: /start -> Главное меню -> "Записаться" -> Выбор услуги -> Выбор мастера -> Выбор даты -> Выбор времени (с учетом занятости) -> Ввод имени -> Ввод телефона -> Ввод комментария -> Подтверждение.
* **Напоминания (Reminders Flow)**: Планировщик проверяет активные записи и отправляет уведомления за 24ч, 2ч и 10 мин до визита.
* **Отмена (Cancellation Flow)**: "Мои записи" -> Выбор записи -> Нажать "Отменить". Проверка времени: если осталось >24ч — статус меняется на Cancelled. Если <24ч — сообщение: "До визита менее суток, для отмены свяжитесь с администратором [Контакт]".
* **Уведомления админу (Admin Flow)**: При успешной записи бот отправляет админу детали (кто, когда, на что).

## 6. API & Integrations
* **Telegram Bot API**: библиотека `aiogram` (версия 3.x).
* **База данных**: Google Sheets API (библиотеки `gspread` / `google-auth`). Авторизация через Service Account (JSON).

## 7. Secrets & Env vars
`.env` файл будет содержать:
`BOT_TOKEN` — токен бота.
`ADMIN_CHAT_ID` — ID чата администратора.
`SHEETS_CREDENTIALS_PATH` — путь к JSON-файлу Google Service Account.
`TIMEZONE` — "Europe/Moscow" (т.к. Краснодар).
`SPREADSHEET_ID` — ID гугл-таблицы.

## 8. Hosting & Run instructions
* **Запуск MVP**: `polling` (`python main.py`).
* **Локальная разработка**: запуск через терминал с установленными зависимостями из `requirements.txt`.
* **Хостинг**: Репозиторий на GitHub.

## 9. Security & Privacy Notes (152-ФЗ)
Перед стартом пользователь дает согласие на обработку персональных данных.
Номера телефонов хранятся только в защищенной таблице Google Sheets. Service Account JSON должен быть строго в `.gitignore`.

## 10. Acceptance criteria (Критерии приемки MVP)
1. Бот корректно запускается и встречает пользователя меню.
2. Прохождение полного цикла записи с попаданием в Google Sheets (без конфликтов времени).
3. Администратор получает push-уведомление в ТГ о новой записи.
4. Планировщик корректно шлёт напоминания.
5. Отмена работает с правилом 24 часов.
